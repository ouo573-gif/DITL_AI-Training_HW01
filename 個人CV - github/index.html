SHORT-TERM

學習目標

  • 強化研究實務基本功：訪談設計、招募、訪談執行、資料整理與初步分析
  • 讓第一學期 UI/UX 專案變成可展示的作品：補齊研究脈絡、原型完整度與驗證結果
  • 建立 Vibe coding 的可用能力：能把原型做成可測試的互動版本

學習行動

  • 研究能力：跟進更多訪談觀摩與實際執行，累積不同情境下的訪談經驗；建立訪談工作流：訪綱版本管理、招募紀錄、摘要模板、初步編碼方式
  • UI/UX 專案深化：重新整理專案敘事：問題定義、關鍵洞察、設計決策、迭代前後差異；以 Vibe coding 補齊主要互動流程與介面細節，完成可操作的 demo；規劃小型測試並回收回饋，形成至少一輪迭代與修正記錄
  • 技術能力：選定一個前端框架與基本技術範圍，完成最小可行作品並可部署展示；建立開發筆記與元件化習慣，確保後續能快速擴充
訪談與研究資料包 v1 UI/UX 專案作品化版本 Vibe coding 基礎作品

MEDIUM-TERM

學習目標

  • 論文主題聚焦並建立可持續累積的研究資料來源
  • 完成一輪研究型專案的實證流程，讓資料量足以支撐洞見與論述
  • 提升英文能力到能在研討會情境下清楚口語表達與簡短問答
  • 把 AI 從工具使用提升到研究流程的一部分，能被記錄、比較與評估

學習行動

  • 論文主題聚焦：以研究室方向與實踐場域收斂研究問題 v1，明確定義研究對象、情境與任務；建立研究路線圖：要收集的資料、要產出的設計工具、要做的比較或實驗設計

論文主題與研究問題 v1＋研究路線圖 一輪完整研究資料包 TOEFL 成績更新＋研討會口語表達練習材料

LONG-TERM

學習目標

  • 參與國際研討會，完成投稿、審查回覆與修稿
  • 完成論文與可公開作品集頁面，具備可被引用的研究輸出

學習行動

  • 研究深化與系列實驗：在確定的研究主題下建立清楚的比較設計或系列研究；以可重複的方法累積足量資料，形成可被檢視的洞見與限制條件；完成論文主要章節草稿，並逐步補齊圖表、方法描述與討論
  • 投稿與發表：以國際研討會為目標，完成摘要、全文、圖表與簡報；依審查意見進行修正並形成第二版論述
